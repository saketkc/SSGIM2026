Multiple hypothesis testing: A primer

The problem of multiple testing

Say you are developing a diagnostic panel and have 500 candidate genes that might be associated with cancer risk. You want to know which genes are truly associated and which associations arose by chance.

For each gene you measure expression in cases vs. controls and run a statistical test (say a t-test).

One test at α = 0.05 carries a 5% chance of a false positive (Type I error). With 500 independent tests at α = 0.05:

\[\Pr(\text{at least one false positive}) = 1 - (1 - 0.05)^{500} = 1 - (0.95)^{500} \approx 0.99999\]

So you will end up something significant (by chance) even when nothing really is.

Why does this happen?

Think of each null gene test as a biased coin that lands heads (a false positive) with probability α = 0.05. One flip rarely comes up heads. But if you flip 500 such coins once each you expect \(500 \times 0.05 = 25\) heads. So what was a rare head per-coin event becomes common once you have enough coins.

So if all 500 genes were truly null, you would still expect ~25 to cross p < 0.05 by chance.

Now suppose the panel flags 30 genes and only a handful of the 500 are genuinely associated. About 25 of the 30 are chance hits and only ~5 are real, an 83% false discovery rate. Reporting all 30 would give patients misleading results and send everyone chasing false leads.

This is the multiple testing problem.

Two error rates

Family-Wise Error Rate (FWER) is the probability of making even one false positive across the whole family of tests. Controlling it is conservative: you make few false calls but miss real effects. Bonferroni controls FWER.

False Discovery Rate (FDR) is the expected fraction of your discoveries that are false. Controlling it is more permissive: you tolerate, say, 5% false calls among the ones you report, and in exchange you recover more real effects. Benjamini-Hochberg controls FDR.

Clinical setting

The Benjamini-Hochberg procedure

The algorithm

  1. Sort all p-values from smallest to largest
  2. Number them: i = 1, 2, 3, …, m (where m is total number of tests)
  3. Calculate threshold for each test: (i/m) × α (where α is your desired FDR, usually 0.05)
  4. Find the largest i where: p-value ≤ (i/m) × α
  5. Reject all hypotheses from 1 to i

A worked simulation:

#  100 tests, 20 are truly significant
set.seed(123)
n_tests <- 100
n_true <- 20
n_null <- n_tests - n_true

# Generate p-values
# True alternatives: small p-values (from exponential distribution)
p_true <- rbeta(n_true, 0.5, 10)
# True nulls: uniform p-values
p_null <- runif(n_null, 0, 1)

# Combine and create dataset
p_values <- c(p_true, p_null)
true_status <- c(rep("True Effect", n_true), rep("No Effect", n_null))

test_data <- data.frame(
  test_id = 1:n_tests,
  p_value = p_values,
  truth = true_status
) %>%
  arrange(p_value) %>%
  mutate(
    rank = row_number(),
    bh_threshold = (rank / n_tests) * 0.05,
    bonf_threshold = 0.05 / n_tests,
    significant_bh = p_value <= bh_threshold,
    significant_bonf = p_value <= bonf_threshold
  )

# Find the BH cutoff
bh_cutoff <- max(which(test_data$significant_bh))

# Create the visualization
ggplot(test_data, aes(x = rank, y = p_value)) +
  # BH threshold line
  geom_line(aes(y = bh_threshold), color = "blue", linewidth = 1, linetype = "solid") +
  # Bonferroni threshold line
  geom_hline(yintercept = 0.05 / n_tests, color = "orange", linewidth = 1, linetype = "dashed") +
  # Alpha line

  # Points colored by truth

Why does this work?

The ranking creates protection

True effects tend to have small p-values, so they cluster at the low ranks, while a true null gives a Uniform(0,1) p-value that could land anywhere. Sorting groups most real effects near the front. The threshold at rank \(i\) is \((i/m)\alpha\), rising by only \(\alpha/m\) per step. Because null p-values are uniform, the chance that any one null falls under the threshold is just the threshold itself, so across all \(m_0\) nulls the expected number of false rejections stays bounded by \(m_0\alpha/m \le \alpha\) times the number of discoveries. That is what caps the FDR while the early-ranking real effects still get rejected.

The mathematical intuition

Why the (i/m) × α formula?

The threshold is \(p_{(i)} \le (i/m)\,\alpha\), where:

  • \(i\) = the rank of this p-value (1 = smallest)
  • \(m\) = total number of tests
  • \(\alpha\) = target FDR (e.g. 0.05)

As you accept more tests (larger \(i\)) the threshold relaxes. That is safe because true effects cluster at small p-values and rank early, and the ratio \(i/m\) grows slowly, so the procedure spends its error budget gradually rather than all at once.

Why does this control FDR?

The full proof is technical. But an intuition could work: Suppose there are \(m_0\) true nulls and \(m_1\) true alternatives. Under a true null a p-value is Uniform(0,1), while true alternatives concentrate near 0. Sorting all p-values together pushes the nulls toward the high ranks, where the relaxed threshold still rejects only a controlled fraction of them. Working through the algebra, the expected fraction of false discoveries is at most \(\alpha \, (m_0/m) \le \alpha\).

One consequence: the more true effects there are, the better BH does, because the alternatives fill the low ranks and crowd the nulls out.

Example: Gene expression analysis

Consider a larger example: 1000 genes, 50 of them truly differentially expressed.

# Simulate gene expression study
set.seed(456)
n_genes <- 1000
n_true_de <- 50 # 50 genes truly differentially expressed

# Simulate p-values
p_values_real <- c(
  rbeta(n_true_de, 0.3, 15), # True DE genes: small p-values
  runif(n_genes - n_true_de, 0, 1) # Non-DE genes: uniform
)

gene_data <- data.frame(
  gene_id = paste0("Gene_", 1:n_genes),
  p_value = p_values_real,
  truth = c(rep("DE", n_true_de), rep("Not DE", n_genes - n_true_de))
) %>%
  arrange(p_value) %>%
  mutate(
    rank = row_number(),
    # Apply different methods
    bh_threshold = (rank / n_genes) * 0.05,
    bh_significant = p_value <= bh_threshold,
    bonf_significant = p_value <= (0.05 / n_genes),
    unadjusted_significant = p_value <= 0.05
  )

# Calculate discoveries and false discoveries for each method
calc_performance <- function(data, sig_col, truth_col = "truth") {
  true_pos <- sum(data[[sig_col]] & data[[truth_col]] == "DE")
  false_pos <- sum(data[[sig_col]] & data[[truth_col]] == "Not DE")
  true_neg <- sum(!data[[sig_col]] & data[[truth_col]] == "Not DE")
  false_neg <- sum(!data[[sig_col]] & data[[truth_col]] == "DE")

  total_discoveries <- true_pos + false_pos
  fdr_actual <- if (total_discoveries > 0) false_pos / total_discoveries else 0

  data.frame(
    True_Positives = true_pos,
    False_Positives = false_pos,
    Total_Discoveries = total_discoveries,
    FDR_Actual = round(fdr_actual, 3),
    Power = round(true_pos / 50, 3) # 50 true DE genes
  )
}

performance <- data.frame(
  Method = c("Unadjusted (α=0.05)", "Bonferroni", "Benjamini-Hochberg"),
  rbind(
    calc_performance(gene_data, "unadjusted_significant"),
    calc_performance(gene_data, "bonf_significant"),
    calc_performance(gene_data, "bh_significant")
  )
)

The realised false-discovery proportion for BH in this run is 0.067. FDR is controlled in expectation (E[V/R] ≤ α), not on every single run, so this one number can land above or below 0.05; averaged over many simulations it settles at or under the target.
